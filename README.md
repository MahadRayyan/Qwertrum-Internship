# 🏢 Qwertrum — Data Analytics Internship

A one-month remote data analytics internship focused on end-to-end sales data analysis — from raw data cleaning and SQL querying through to time series forecasting, A/B testing, interactive Power BI dashboards, and business-ready PowerPoint presentations delivered to stakeholders.

---

## ⚡ Technologies

- Python
- Pandas
- NumPy
- SQL
- Power BI
- Microsoft Excel
- PowerPoint

---

## 🚀 What I Worked On

- **Data Cleaning** — handled missing values, duplicates, inconsistent formatting, and data type mismatches across raw sales datasets using Python (Pandas) and Excel, producing clean, analysis-ready tables
- **SQL Querying** — wrote queries to extract, filter, aggregate, and join sales records from relational databases to answer specific business questions
- **Time Series Analysis** — analyzed sales trends over time, identifying seasonal patterns, growth periods, and anomalies in revenue data
- **A/B Testing** — designed and evaluated A/B experiments on sales strategies or campaigns, comparing group performance using statistical methods to support data-driven decisions
- **Power BI Dashboard** — built an interactive sales dashboard with slicers, KPIs, and visualizations tracking revenue, product performance, and regional trends
- **Business Presentation** — compiled findings and insights into a clean PowerPoint deck delivered to business owners, translating technical analysis into clear, actionable recommendations

---

## 🧠 The Experience

Coming into Qwertrum, I knew the theory — cleaning data, building dashboards, running analyses. The internship was where I learned what that actually looks like in a professional setting with real, messy sales data and real stakeholders waiting on answers.

The most valuable part wasn't any single technique — it was learning to move through a full data pipeline from end to end: raw data → cleaned data → analysis → visualization → business story. Every step feeds the next, and if the cleaning is sloppy, everything downstream suffers.

A/B testing was the sharpest learning curve. Setting up a fair experiment, choosing the right statistical test, and then explaining the result to a non-technical audience in a PowerPoint slide — that's a completely different skill from just running the numbers. By the end, I had built something I could actually hand to a business owner and have them act on it.
